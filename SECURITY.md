# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in bagsball, please report it responsibly.

### Where to Report

**DO NOT** open a public issue for security vulnerabilities.

Instead:
1. Email: [Create a private security advisory on GitHub]
2. Provide detailed information about the vulnerability
3. Include steps to reproduce if possible

### What to Include

- Description of the vulnerability
- Potential impact
- Steps to reproduce
- Suggested fix (if any)
- Your contact information

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Fix Timeline**: Depends on severity

## Scope

### In Scope

- Dashboard (apps/web)
- Worker/verification scripts (apps/worker)
- Shared utilities (packages/shared)
- Configuration and deployment scripts

### Out of Scope

- DividendsBot (operated by Bags team)
- BagsAMM (operated by Bags team)
- Bags platform infrastructure
- Solana blockchain itself
- Third-party dependencies (report to respective projects)

## Security Considerations

### What bagsball Controls

- Frontend dashboard code
- Verification scripts
- Documentation

### What bagsball Does NOT Control

- DividendsBot operations
- BagsAMM operations
- On-chain token program (if launched)
- User wallets and private keys

### User Responsibilities

Users are responsible for:
- Securing their wallet private keys
- Verifying website URLs
- Understanding meme coin risks
- Doing their own research (DYOR)

## Best Practices for Users

1. **Verify URLs**: Always check you're on the official site
2. **Wallet Security**: Never share seed phrases
3. **HTTPS**: Ensure secure connection
4. **Phishing**: Be wary of fake accounts/websites
5. **Smart Contracts**: Verify contract addresses

## Known Limitations

- bagsball is a meme coin with inherent market risks
- No control over external bot operations
- Dependent on Bags infrastructure
- Subject to Solana network conditions

## Disclosure Policy

- We follow responsible disclosure
- Credit given to reporters (unless anonymous)
- Public disclosure after fix is deployed
- Security advisories published as needed

## Updates

This security policy may be updated. Check back regularly for changes.
